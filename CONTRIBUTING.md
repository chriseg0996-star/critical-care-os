# Contributing

## Development rules

1. Keep modules focused.
2. Do not duplicate global rules across modules unless clinically necessary.
3. Prefer actionable ICU behavior over generic instructions.
4. Include examples for any new documentation module.
5. Update `CHANGELOG.md` with every meaningful change.

## Style

- Spanish for clinical documentation behavior.
- English for repository architecture and technical conventions.
- Avoid vague wording.
- Avoid bloated prompts.
